# Merchant Pack — onboarding & acceptance docs for Phase 5

The operator-facing documents for taking one real merchant from "yes" to a
running production storefront. Companion to the engineering runbook
([MERCHANT-DEPLOYMENT-V1.md](../MERCHANT-DEPLOYMENT-V1.md)) — that document
is *how to deploy*; these are *how to onboard and accept*.

| Doc | Use it when |
|---|---|
| [ONBOARDING-CHECKLIST.md](./ONBOARDING-CHECKLIST.md) | A merchant said yes — collect what's needed, track the deployment |
| [FIRST-ORDER-WALKTHROUGH.md](./FIRST-ORDER-WALKTHROUGH.md) | Go-live day — the acceptance walk, merchant-facing version |

Sales material (demo script, packages/pricing) is maintained separately and
is not part of this repository.

Ground rules carried over from the platform docs:

- The demo portal (`bp-cms.doehpos.com`) is sandbox-only and resets nightly —
  demo freely, promise nothing that isn't live.
- Keys are minted on the DOEH developer portal and handed over out-of-band —
  never through chat, email, or these documents.
